## Setup guide

Dynamics 365 Customer Engagement (Dataverse) authenticates through Microsoft Entra
(Azure AD) using the OAuth 2.0 authorization code flow.

1. Register an application in the [Microsoft Entra admin center](https://entra.microsoft.com).
2. Add the Dynamics CRM delegated permission (`user_impersonation`).
3. Create the Nango integration and provide:
   - `tenantId` — your Azure AD tenant id.
   - `instanceUrl` — your Dataverse host, e.g. `org.crm.dynamics.com`.
4. Configure the integration scopes with the per-instance resource scope, e.g.
   `https://org.crm.dynamics.com/user_impersonation offline_access`. Nango does not
   interpolate the connection's `instanceUrl` into the authorize-URL scope, so this
   must be supplied as the integration's scopes at configuration time.

<Tip>Need help getting started? Get help in the [community](https://nango.dev/slack).</Tip>

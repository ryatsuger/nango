## Setup guide

Suger exposes its marketplace data through a remote MCP server. The OAuth authorization
server and the MCP server are the same per-environment API host:

| Environment | API host (`host`) |
| - | - |
| Production | `apiv2.suger.cloud` |
| Staging | `apiv2.stag.suger.cloud` |
| Development | `apiv2.dev.suger.cloud` |

The host serves both OAuth (`/oauth2/authorize`, `/oauth2/token`) and the MCP endpoint
(`POST /mcp`). Create the integration with the fixed OAuth client id `suger-console` (the
provider uses `client_registration: static`), and supply the `host` connection
configuration value for the environment you are connecting to.

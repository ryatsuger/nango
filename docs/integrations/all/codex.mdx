## Setup guide

Codex uses a public OAuth client with PKCE and a device-code flow. Nango requests a user code from OpenAI, you open the verification page and enter that code, then Nango polls until you approve and exchanges the result for tokens.

<Tip>Need help getting started? Get help in the [community](https://nango.dev/slack).</Tip>

## API gotchas

-   This integration uses `auth_mode: OAUTH2_DEVICE_CODE`. The connection is created in two steps (`/oauth/device/{providerConfigKey}/start` then repeated `/oauth/device/{providerConfigKey}/poll`); there is no hosted callback.
-   The OAuth client is public (no client secret). The fixed public `client_id` is supplied by the provider template.
-   Proxy requests require the `ChatGPT-Account-Id` header. Nango extracts it from the `id_token` returned during authorization and stores it on the connection.
